Oracle Complex Event Processing(CEP)
 - 여러 이벤트 소스로부터 발생하는 여러 이벤트를 대상으로 의미 있는 데이터를 실시간으로 추출하는 액션
 - Oracle Continuous Query Language(CQL) 사용
 - 기존 Query는 한번 결과가 나오면 종료
 - CQL은 사용자가 멈출 때까지 계속 구동
 - Oracle 11g부터 탑재 (2007)

 Oracle CEP 특징
Oracle Complex Event Processing (CEP)는 다음과 같은 특징을 가진다.
경량 서버의 CEP 엔진
필터링, 패턴 매칭, 사용자 윈도우(시간 윈도우, 스트림 Row 윈도우 등), 메시지 강화 등 복잡한 이벤트를 처리하기 위한 CEP 주요 기능 제공
Oracle CQL(Continuous Query Language)
프로세싱 언어로써 SQL의 확장판인 Oracle CQL을 기반으로 하여 표준 SQL 사용 가능시 접근이 용이
Real Time JVM, OSGi 기반 경량 서버
높은 Throughput과 서비스 지연을 방지하기 위해 Real Time JVM으로 Oracle JRockit 사용, Equinox OSGi 기반 경량 자바 컨테이너 사용
하이브리드 프로그래밍 모델
자바와 CQL을 유연하게 연동하여 개발하는 모델을 사용하여, 복잡한 이벤트 처리 기술인 CQL Query와 비즈니스 로직 처리를 위한 Java를 조합하여 프로그래밍 가능
재사용 가능한 컴포넌트
CEP 애플리케이션은 재사용 가능한 설정기반 컴포넌트를 사용. CQL 룰 설정은 런 타임에 변경 가능
